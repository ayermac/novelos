# 判死刑机制：违禁扫描

## 最高权限指令

在执行打分前，必须先进行"违禁词汇与套路"扫描。一旦发现以下任何一种情况，**无论其他层面写得多好，直接停止打分流程，将该项赋予 0 分，总分强行压至 50 分以下打回！**

---

## 死刑触发库

### 1. 机械降神 / 强行开挂

触发条件：
- 未经过铺垫的神秘高人（如：扫地大爷、神秘前辈）
- 天降神器
- 系统没有经过任务就直接给出核心答案

**示例**：
> ❌ 就在林默绝望之际，一位白发老者突然出现，轻轻一挥手就化解了危机...

**正确做法**：
- 提前3-5章埋下神秘人物的线索
- 或主角通过努力获得关键能力/道具

---

### 2. AI 重复烂梗

**一章内出现以下任意词汇 ≥ 1 次，即触发死刑**：

| 违禁词汇 | 触发条件 | 替代方案 |
|---------|---------|---------|
| 冷笑 | 出现1次即触发 | 用具体动作：他挑眉、嘴角抽动 |
| 嘴角微扬 | 出现1次即触发 | 他笑了、表情缓和 |
| 眼中闪过寒芒/冷意 | 出现1次即触发 | 目光锐利、眼神阴沉 |
| 深吸/倒吸一口凉气 | 出现1次即触发 | 倒吸一口气、屏住呼吸 |
| 融入夜色/消失在夜色 | 出现1次即触发 | 走进黑暗、身影隐没 |
| 不由得/不禁/忍不住 | 出现1次即触发 | 直接描述动作或心理 |

**检测方法**：
```bash
python3 tools/db.py check_chapter <project> <chapter>
```

---

### 3. 上帝视角泄露

正文中出现原本只存在于数据库或大纲里的元数据：

**违禁词汇**：
- "伏笔已记录"
- "剧情节点"
- "任务完成"
- "系统提示："
- 任何仅供 Planner 使用的破壁词汇

**示例**：
> ❌ 林默心中暗想：这个伏笔我已经记录在案，等待后续触发...

**正确做法**：
- 用角色视角的自然语言
- 主角有自己的疑惑或计划，但不能出现"伏笔"等元数据词汇

---

### 4. 句式类死刑

| 违禁句式 | 问题 | 修改建议 |
|---------|------|---------|
| 不仅...而且...更是... | AI公文感 | 拆成两句独立陈述 |
| 夜色笼罩/夜幕降临 | 烂俗开篇 | 用具体时间或场景 |
| 心中暗想/心道 | 说书人口吻 | 直接描述心理活动 |

---

### 5. 说教类死刑

| 违禁内容 | 问题 |
|---------|------|
| 章节末尾总结人生道理 | AI喜欢在结尾升华 |
| 上帝视角的哲理感慨 | 脱离角色视角 |
| "这让他明白了..." | 机械式总结 |

**示例**：
> ❌ 这让林默明白了一个道理：在这个世界上，只有强者才能生存...

**正确做法**：
- 用角色的行动展示领悟
- 不需要总结性陈述

---

## 死刑处理流程

1. **立即停止打分**
2. **在 `add_review` 的 issues 字段中输出**：
   ```
   触发系统最高红线 [违禁内容]，本章无效，立即重写。
   
   具体问题：
   - 第3段出现"冷笑"
   - 结尾出现人生道理总结
   ```
3. **总分设为 50 分**
4. **pass = false**
5. **跳转到 Step 11 提交退回报告**

---

## 死刑红线检查清单

```
□ 机械降神：是否有未铺垫的神秘人物/神器救场？
□ AI烂梗：是否有"冷笑"、"嘴角微扬"、"眼中闪过寒芒"？
□ 上帝视角：是否有"伏笔已记录"、"剧情节点"等破壁词汇？
□ 烂俗句式：是否有"不仅...而且...更是..."？
□ 说教总结：章节末尾是否有人生道理总结？
```
