# 调度状态机参考

本文档详细说明调度器的状态转换逻辑和决策树。

## 状态优先级

章节状态优先级（从高到低）：

| 优先级 | 状态 | 触发条件 | 调度动作 | 目标 Agent |
|--------|------|----------|----------|------------|
| P0 | blocking | 死循环熔断 / 数据异常 | 暂停流程，发消息给人类 | 人类 |
| P1 | timeout | 任务超时 30 分钟 | 标记为 failed，等待重试 | DB 操作 |
| P2 | revision | 质检退回，有修改意见 | 触发重写 | author |
| P3 | review | 待审核，有草稿 | 触发审校打分 | editor |
| P4 | reviewed | 已通过质检 | 触发最终发布与伏笔回收 | planner |
| P5 | planned | 有创作指令 | 触发新章节创作 | author |
| P6 | empty | 章节表为空或需新章 | 触发撰写下一章指令 | planner |

## 一致性检查决策树

```
开始检查
    │
    ├── 有 revise 任务 ≥ 3 次？
    │       │
    │       └─YES─▶ 【熔断】abandon，通知人类
    │
    ├── 无任务记录？
    │       │
    │       └─YES─▶ trigger，首次触发
    │
    ├── 有 running 任务？
    │       │
    │       ├─YES─▶ 超时 > 30 分钟？
    │       │           │
    │       │           ├─YES─▶ timeout，标记失败
    │       │           │
    │       │           └─NO──▶ skip，等待完成
    │
    ├── 有 failed 任务？
    │       │
    │       ├─YES─▶ 重试次数 < 3？
    │       │           │
    │       │           ├─YES─▶ retry，重试任务
    │       │           │
    │       │           └─NO──▶ abandon，需人工干预
    │
    └─ 只有 completed 任务
            │
            └────▶ trigger，触发下一节点
```

## 熔断机制详解

### 死循环熔断

**触发条件**：同一章节被质检退回 ≥ 3 次

**原因分析**：
- 执笔 Agent 陷入创作盲区，无法理解修改意见
- 大纲设计存在逻辑矛盾
- 数值状态卡点无法突破

**处理方式**：
1. 标记章节为 `blocking` 状态
2. 发送消息通知用户
3. 等待人类介入修改大纲或指令

### 超时熔断

**触发条件**：任务运行超过 30 分钟

**可能原因**：
- Agent 卡死
- LLM API 超时
- 系统资源不足

**处理方式**：
1. 标记任务为 `failed`
2. 下一轮调度自动重试（最多 3 次）

### 重试熔断

**触发条件**：底层错误重试 3 次均失败

**可能原因**：
- 数据库连接问题
- 配置错误
- 系统级 Bug

**处理方式**：
1. 标记章节为 `blocking`
2. 发送消息通知用户
3. 需人工检查日志

```
       │

       │              │              │

```

## 锁机制详解

### 为什么需要锁？

OpenClaw 的 `sessionTarget: "isolated"` 模式意味着每次 Cron 都创建新会话：

```
时间线:
21:00 ─ Cron A 创建会话1，开始调度
21:05 ─ Cron B 创建会话2，也开始调度
       ↓
       两个会话可能同时 spawn 相同任务！
```

### 锁的工作原理

```python
# 使用 O_EXCL | O_CREAT 确保原子性
fd = os.open(LOCK_FILE, os.O_CREAT | os.O_EXCL | os.O_WRONLY)
# 只有一个进程能成功创建文件
# 其他进程会收到 FileExistsError
```

### 锁的清理

- 正常情况：调度结束时 `release_lock()`
- 异常情况：进程崩溃后锁文件残留
- 安全措施：检查锁文件的 PID 是否仍在运行
