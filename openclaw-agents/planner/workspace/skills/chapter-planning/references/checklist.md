# 规划检查清单

## 发布指令前核对

```
□ 状态注入：objective 开头是否已包含最新 chapter_state？
□ 红线自检：key_events 是否有机械降神套路？
□ 逻辑闭环：反派行为是否具备合理动机？
□ 动作指导：是否提醒禁用冷笑/微扬等AI惯用词？
□ 钩子强度：ending_hook 是否制造了悬念或危机？
□ 参数合规：伏笔参数是否为 JSON 数组？顺序正确吗？
□ 大纲调用：是否查询了对应的卷/篇章规划？
```

## 任务完成检查

```
□ 是否已执行 task_complete？
  没有这一步，调度器永远认为任务还在运行！
```

## 设定变化检查

```
□ 本章是否有角色死亡/失踪/觉醒？
  → 如有，是否已执行 update_character？

□ 本章是否有势力关系变化？
  → 如有，是否已执行 update_faction？

□ 本章是否有新角色登场？
  → 是否已在 create_instruction 的 new_characters 中添加？

□ 本章是否有新世界观设定？
  → 是否已执行 add_world_setting？

□ 设定变化是否与伏笔/大纲一致？
```

## 数据校对检查

| 检查项 | 错误级别 | 说明 |
|--------|---------|------|
| 伏笔引用不存在 | ❌ error | plots_to_plant/resolve 引用的伏笔不存在 |
| 伏笔未埋设即兑现 | ❌ error | 兑现的伏笔未在之前章节埋设 |
| 状态卡缺失 | ⚠️ warning | 上一章没有状态卡 |
| 新角色创建 | ℹ️ info | 指令将创建新角色（正常） |
