# 市场报告参数规范

## 命令格式

```bash
python3 tools/db.py add_market_report <date> "<summary>" '<hot_genres>' '<trending_tags>' '<opportunities>' '<risks>' '<advice>'
```

---

## 参数详解

### summary（市场总结）

**一句话总结**当前市场的变局与核心情绪。

**示例**：
```
"传统装逼打脸已疲软，【发疯文学】与【高智商反派】成为读者新宠，情绪价值从'爽'转向'解压/反内卷'。"
```

---

### hot_genres（热门题材）

**具体到微分类**，不要只写"修仙火"、"系统火"。

**JSON 数组格式**：
```json
["规则怪谈+打工人", "反派逆袭+高智商权谋", "真假千金发疯流"]
```

**微分类模式**：
- 题材A + 题材B（缝合）
- 题材 + 情绪标签（发疯流、躺平流）

---

### trending_tags（趋势标签）

**核心爽点/金手指机制**。

**JSON 数组格式**：
```json
["不圣母", "系统漏洞利用", "反套路", "听劝变强"]
```

---

### opportunities（机会）

**蓝海/微创新组合**。

**JSON 数组格式**，每项包含：
- `type`: 机会类型（"蓝海"、"微创新蓝海"、"缝合创新"）
- `genre`: 题材组合
- `evidence`: 数据证据

**示例**：
```json
[
  {"type": "微创新蓝海", "genre": "修仙+反向内卷", "evidence": "读者对苦大仇深的升级流厌倦，轻松躺平流数据暴涨"},
  {"type": "缝合创新", "genre": "废土+模拟经营", "evidence": "近期同类无脑囤物资文看疲劳，经营类留存率高"}
]
```

---

### risks（风险）⚠️ 强制填入

**毒点预警**，必须填写。

**JSON 数组格式**，每项包含：
- `type`: 风险类型（"毒点预警"、"过热预警"、"政策风险"）
- `genre`: 问题题材
- `evidence**: 数据证据

**示例**：
```json
[
  {"type": "毒点预警", "genre": "套路化降智反派", "evidence": "多本新书因反派无脑挑衅被读者狂喷弃书"},
  {"type": "过热预警", "genre": "发疯文学", "evidence": "近期同质化严重，读者开始审美疲劳"}
]
```

---

### advice（建议）

**给总编的实操设定建议**。

**JSON 数组格式**：
```json
[
  "建议开局第1章直接撕破脸，不要憋屈",
  "给反派设定合理的利益动机，甚至比主角更懂法",
  "主角开局必须处于极致负债/绝境状态"
]
```

```bash
python3 tools/db.py add_market_report \
  "2026-04-04" \
  "传统装逼打脸已疲软，【发疯文学】与【高智商反派】成为读者新宠，情绪价值从'爽'转向'解压/反内卷'。" \
  '["规则怪谈+打工人", "反派逆袭+高智商权谋"]' \
  '["不圣母", "系统漏洞利用", "反套路"]' \
  '[{"type":"微创新蓝海","genre":"修仙+反向内卷","evidence":"读者对苦大仇深的升级流厌倦，轻松躺平流数据暴涨"}]' \
  '[{"type":"毒点预警","genre":"套路化降智反派","evidence":"多本新书因反派无脑挑衅被读者狂喷弃书"}]' \
  '["建议开局第1章直接撕破脸，不要憋屈","给反派设定合理的利益动机，甚至比主角更懂法"]'
```

## 格式检查

```text
□ summary 是一句话总结吗？
□ hot_genres 是具体到微分类的 JSON 数组吗？
□ trending_tags 是核心爽点/金手指的 JSON 数组吗？
□ opportunities 包含 type、genre、evidence 吗？
□ risks 有毒点预警吗？（强制！）
□ advice 是可操作的建议吗？
□ JSON 格式正确吗？（引号、逗号）
```
