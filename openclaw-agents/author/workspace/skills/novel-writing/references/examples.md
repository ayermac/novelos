## 二、伏笔正确兑现示例

### 伏笔设定

```
S004 - 林默的非理性软肋
描述：林默对妻子赵婉清有极强的保护欲，当赵婉清被羞辱或威胁时会失去理智。
      这是林默唯一的非理性弱点。
```

### ✅ 正确的伏笔兑现方式

```markdown
赵国栋冷笑着转向赵婉清："婉清，你看看这个废物。他连给你买件像样的衣服都要跟我伸手。"

林默的眼神瞬间变了。

那不是愤怒。那是某种更深、更冷的东西——像是一头被逼入绝境的狼，露出了獠牙。

"你可以说我。"林默的声音压得很低，每个字都像是从牙缝里挤出来的，"但别动她。"

赵婉清从未见过林默这样的眼神。那是她的丈夫，但又不完全是。
那个眼神里有一种近乎疯狂的、非理性的保护欲。

这是他唯一的软肋。
```

**为什么这样写能通过验证：**
- 包含"林默"、"赵婉清"（实体）
- 包含"保护欲"、"非理性"、"软肋"（关键词+实体）
- 伏笔在剧情中自然体现，不是强行植入

### ❌ 错误的伏笔兑现方式

```markdown
林默对赵婉清有极强的保护欲（这是他的非理性软肋S004）。
```

**为什么这样写是错误的：**
- 强行植入关键词
- 在内容中写入伏笔代码（S004）
- 破坏了叙事流畅性

## 三、伏笔触发对象验证

### 核心规则

**伏笔说"A被羞辱/威胁" → 必须是A本人被直接羞辱/威胁**

### 错误理解

- ❌ B被羞辱，A被牵连
- ❌ A因为B的事情而受影响
- ❌ 隐喻性的、间接的羞辱
- ❌ 羞辱与A相关的事物（如"继承权"）

### 正确理解

- ✅ A本人被直接羞辱："你就是个废物"
- ✅ A本人被直接威胁："你要是不X，就Y"
- ✅ A本人的利益、尊严、人身安全被直接攻击

## 四、触发对象验证示例

**伏笔：S004 - "当赵婉清被羞辱或威胁时，林默会失去理智"**

### ❌ 错误兑现（第一次）

```
赵国栋威胁继承权："赵家的产业，你别想继承一分钱"
↓
这是威胁"继承权"，不是威胁"赵婉清本人"
↓
林默愤怒（因为自己被羞辱）
```

### ❌ 错误兑现（第二次）

```
赵国栋说："你让婉清以后怎么见人"
↓
这是羞辱林默"让赵婉清丢脸"，不是羞辱赵婉清本人
↓
林默愤怒（因为自己让妻子丢脸）
```

### ✅ 正确兑现

```
赵国栋直接羞辱赵婉清："婉清，你看看你这个丈夫，连给你买件像样的衣服都要跟我伸手。
你嫁给他三年，他给你挣过一分钱吗？你是我赵国栋的女儿，嫁给这么一个废物，
你以为你在帮他？你是在毁你自己。"
↓
这是直接羞辱赵婉清本人
↓
赵婉清的脸色刷地变白，眼眶泛红
↓
林默的眼神瞬间变了——那是一种近乎疯狂的、非理性的保护欲
```

## 五、数值创作基准示例

### 状态卡读取后注入创作上下文

```
【创作基准 - 来自第2章结束状态】
系统等级：Lv1
建造点数：剩余 5/10
资源：水✅ 食料❌
辐射：18%
已解锁：主播光环、图纸系统Lv1

--- 创作指令 ---
[从 instruction 表读取的实际指令]
```

### 数值消耗规则

```
❌ 错误：消耗3点建造点数后还剩8点（假设初始10点）
✅ 正确：从状态卡读取 current=5，消耗3后剩余2点

❌ 错误：主角突然解锁了"高级建造"功能
✅ 正确：从状态卡读取 unlocked_features，不在列表里就不能用
```

### 常见数值维度

| 维度 | 说明 |
|------|------|
| 系统等级/修为 | 当前主角等级/境界 |
| 资源点数 | 建造点数/灵力值/金币等 |
| 资源存量 | 水/食物/物资库存 |
| 环境数值 | 辐射/天气/危险等级 |
| 人口/势力 | 角色数量/阵营关系 |
| 已解锁内容 | 功能/技能/建筑 |
