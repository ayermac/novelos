# 日报输出模板

## 完整模板

```markdown
# 🏭 网文工厂运营日报 · {YYYY-MM-DD}

## 📊 产能与质量大盘
- **运行项目**：X 个
- **今日产出**：X 章（新增 X 字）
- **质量均分**：XX 分 *(注：低于90分说明勉强过线)*
- **退回返工**：今日共发生 X 次质检退回操作

---

## 📖 项目运转详情

### 📚 《{书名}》 (项目ID: {project_id})
- **整体进度**：{已完成章数} / {计划总章数} ({进度百分比}%)
- **最新进展**：第 {X} 章已通过质检并发布，得分为 {XX} 分。
- **卡点状态**：*(例如：第 {X} 章目前处于 revision 状态，执笔正在重写)*
- **伏笔债务**：当前有 {X} 条待兑现伏笔（其中超期警报 {X} 条）。

---

## 🚨 异常与熔断预警
*(若无则写"✅ 系统运转良好，无阻塞环节")*

- ⚠️ **重写死锁预警**：*(例如：第 5 章已被质检退回 2 次，请关注是否需要修改策划大纲)*
- ❌ **任务执行失败**：*(例如：调度器报告 `novel_001` 的 author Agent 发生工具调用超时)*
- 💣 **伏笔烂尾风险**：*(例如：长期伏笔 L001 已超过 80 章未兑现，请提醒总编干预)*

---
*下次例行汇报：明日 20:00*
```

---

## 数据字段说明

| 字段 | 来源命令 | 说明 |
|------|---------|------|
| 运行项目 | `projects` | 项目总数 |
| 今日产出 | `stats` | 新增章节数和字数 |
| 质量均分 | `stats` | 平均质检分数 |
| 退回返工 | `chapters` | revision 状态数量 |
| 整体进度 | `chapters` | 已完成/计划总数 |
| 卡点状态 | `chapters` | revision 状态的章节 |
| 伏笔债务 | `pending_plots` | 待兑现伏笔数 |
| 异常预警 | `task_list` | failed 任务、重试次数 |

---

## 预警级别

| 级别 | 图标 | 触发条件 |
|------|------|---------|
| 重写死锁 | ⚠️ | 同一章节 revision ≥ 2 次 |
| 任务失败 | ❌ | task_list 中有 failed 任务 |
| 伏笔烂尾 | 💣 | 长期伏笔超过计划兑现章节 80% |
| 质量警告 | ⚡ | 质量均分 < 85 分 |
| 产能预警 | 📉 | 今日产出 < 计划产出 50% |
