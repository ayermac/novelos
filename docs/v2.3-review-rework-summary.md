# v2.3 Review Rework Summary

## Overview
This document summarizes the v2.3 review rework that fixed 2 critical issues found during review.

## Issues Fixed

### R1 [P1]: Package Handler Loading Failure Must NOT Fallback to Legacy Class

**Problem**: 
When a skill has `package` configured, if `_load_skill_from_package()` failed (handler missing, entry_class not found, not BaseSkill subclass), the code would fallback to legacy `_get_skill_class()` from whitelist. This masked package corruption and violated v2.3's requirement that package handler is the primary loading mechanism.

**Solution**: 
- Modified `SkillRegistry.get_skill()` to treat package configuration as the ONLY loading path
- If `package` is configured, failure to load from package handler returns `None` immediately
- Only skills WITHOUT `package` configuration can use legacy whitelist loading
- Added clear error messages for package loading failures

**Files Modified**:
- `novel_factory/skills/registry.py` (lines 208-246)

**Key Changes**:
```python
# v2.3: If package is configured, it's the ONLY loading path (no fallback)
package_path = skill_config.get("package")
if package_path:
    # Package is configured - must load from package handler
    if not manifest or not manifest.package:
        logger.error(f"Skill {skill_id} has package path but no package manifest")
        return None
    
    skill_class = self._load_skill_from_package(package_path, manifest.package.entry_class)
    if not skill_class:
        logger.error(f"Failed to load skill class from package for {skill_id}")
        return None
    
    # ... instantiate and return
    return skill_instance

# v2.2/v2.1: Legacy loading (only for skills without package)
# ... legacy whitelist loading
```

### R2 [P2]: skills test JSON Error Paths Must Use Unified Envelope

**Problem**: 
1. `skills test --json` without skill_id returned plain text error instead of JSON envelope
2. `skills test unknown-skill --json` had `data: null` instead of `data: {}`
3. Inconsistent JSON output format across error paths

**Solution**: 
- Modified `cmd_skill_test()` to check for `--json` flag before printing error
- Ensured all JSON outputs use `{ok, error, data}` envelope format
- Guaranteed `data` is always a dict `{}`, never `null`

**Files Modified**:
- `novel_factory/cli.py` (lines 1140-1160)

**Key Changes**:
```python
# Test single skill
if not skill_id:
    use_json = getattr(args, "json", False)
    if use_json:
        print(json.dumps({
            "ok": False,
            "error": "skill_id is required (or use --all)",
            "data": {}
        }, ensure_ascii=False))
    else:
        print("Error: skill_id is required (or use --all)")
    sys.exit(1)

# Ensure data is always a dict, never null
data = result.get("data")
if data is None:
    data = {}

envelope = {
    "ok": result.get("ok", False),
    "error": result.get("error") if not result.get("ok") else None,
    "data": data
}
```
