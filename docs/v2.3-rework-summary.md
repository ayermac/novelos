# v2.3 Rework Summary

## Overview
This document summarizes the v2.3 rework that fixed 3 specification-level issues (R1-R3).

## Issues Fixed

### R1 [P1]: Registry Must Load Skill Class from Package Handler
**Problem**: Registry was only reading manifest.class_name and getting class via whitelist, not actually loading from package handler.

**Solution**: 
- Modified `SkillRegistry.get_skill()` to prioritize loading from package handler when package config exists
- Added `_load_skill_from_package()` method that uses `importlib.import_module()` to dynamically load skill class
- Implemented security validation to reject:
  - Absolute paths
  - Directory traversal (`../`)
  - Non-BaseSkill subclasses

**Files Modified**:
- `novel_factory/skills/registry.py`

**Tests Added**:
- `tests/test_v2_3_rework.py::TestR1RegistryLoadsFromPackageHandler` (8 tests)

### R2 [P1]: pyproject.toml Must Package skill_packages Resources
**Problem**: pyproject.toml was not including skill_packages directory in package data.

**Solution**:
- Added `skill_packages/**/*` to `[tool.setuptools.package-data]` section

**Files Modified**:
- `pyproject.toml`

**Tests Added**:
- `tests/test_v2_3_rework.py::TestR2PyprojectPackagesSkillPackages` (3 tests)

### R3 [P2]: skills test JSON Envelope Must Be Unified
**Problem**: `skills test` JSON output was not following the `{ok, error, data}` envelope format.

**Solution**:
- Modified `cmd_skill_test()` in cli.py to ensure envelope format
- Modified `cmd_skill_validate()` to use envelope format
- Fixed all skill-related CLI commands to use envelope format

**Files Modified**:
- `novel_factory/cli.py`

**Tests Added**:
- `tests/test_v2_3_rework.py::TestR3JSONEnvelope` (7 tests)
- `tests/test_v2_3_rework.py::TestR3Integration` (1 test)

## Security Enhancements

### Path Validation
- Absolute paths are rejected
- Directory traversal (`../`) is rejected
- Only relative paths under `novel_factory/skill_packages/<name>/` are allowed

### Class Validation
- Loaded class must be a subclass of `BaseSkill`
- Invalid classes are rejected with None return
